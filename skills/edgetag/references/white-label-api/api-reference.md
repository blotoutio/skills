# White-Label API Reference — Complete Endpoints

Base URL: `https://api.edgetag.io/v1` (production) or `https://api-sandbox.edgetag.io/v1` (sandbox)

All endpoints require:

- `Authorization: Bearer {access_token}` — OAuth 2.0 access token or personal access token
- `Team-Id: {team_id}` — required on all endpoints **except** `GET /user/me`

The access token can come from either:

- **OAuth 2.0 flow** — customer authenticates with their own account (see below)
- **Personal access token** — generated by EdgeTag support ([support@edgetag.io](mailto:support@edgetag.io)). Simpler but all customers share your single account. See [README.md](README.md) § Authentication for trade-offs.

## OAuth 2.0

### 1. Create an OAuth App

Register at `https://app.edgetag.io` (or sandbox) → Team menu → OAuth Apps.

Required fields:

- **App Name** — your product name
- **Description** — what the app does (speeds up approval)
- **Website** — your product URL
- **Support Email** — dedicated support address
- **Privacy Policy** — URL to your privacy policy
- **Terms of Service** — URL to your ToS
- **Origin URIs** — URLs where "Login with EdgeTag" will render
- **Redirect URIs** — URLs EdgeTag redirects to after OAuth

On save, you receive a `client_id` and `client_secret`. **Save the secret immediately** — it cannot be viewed again (only rolled/regenerated).

Apps enter a **pending review** state. EdgeTag notifies via email when approved.

### 2. Create Authorization Redirect

Redirect users to EdgeTag's login page:

```
https://app.edgetag.io/oauth/app/login
  ?client_id={client_id}
  &redirect_uri={redirect_uri}
  &scope=full
  &response_type=code
  &code_challenge={code_challenge}
  &code_challenge_method=S256
  &state={optional_base64_state}
```

For sandbox, use `https://app-sandbox.edgetag.io/oauth/app/login`.

| Parameter               | Required | Description                                           |
| ----------------------- | -------- | ----------------------------------------------------- |
| `client_id`             | Yes      | From OAuth app creation                               |
| `redirect_uri`          | Yes      | Must match a registered redirect URI                  |
| `scope`                 | Yes      | `full` (only option currently)                        |
| `response_type`         | Yes      | `code` (only option currently)                        |
| `code_challenge`        | Yes      | SHA-256 hash of a verifier string, base64url-encoded  |
| `code_challenge_method` | Yes      | `S256` (only option currently)                        |
| `state`                 | No       | Base64-encoded custom data preserved through the flow |

#### Generating code_challenge

```javascript
const crypto = require('crypto')

// Generate and store the verifier (you'll need it later)
const verifier = crypto.randomBytes(32).toString('base64url')

// Hash it for the challenge
const codeChallenge = crypto
  .createHash('sha256')
  .update(verifier)
  .digest()
  .toString('base64url')
```

#### Redirect example

```javascript
const params = new URLSearchParams({
  client_id: clientId,
  redirect_uri: 'https://yourapp.com/callback',
  scope: 'full',
  response_type: 'code',
  code_challenge: codeChallenge,
  code_challenge_method: 'S256',
})

window.location.href = `https://app.edgetag.io/oauth/app/login?${params}`
```

### 3. Exchange Code for Access Token

After redirect, extract the `code` from the callback URL and exchange it:

```
POST https://api.edgetag.io/v1/oauth-app/client/token
Content-Type: application/x-www-form-urlencoded
Authorization: Basic {base64(client_id:client_secret)}
```

| Field           | Value                                      |
| --------------- | ------------------------------------------ |
| `code`          | From the redirect callback URL             |
| `code_verifier` | The **plain-text** verifier (not the hash) |
| `grant_type`    | `authorization_code`                       |

#### JavaScript

```javascript
const code = new URL(window.location.href).searchParams.get('code')

const body = new URLSearchParams({
  code,
  code_verifier: verifier, // plain text, NOT the hashed challenge
  grant_type: 'authorization_code',
})

const response = await fetch(
  'https://api.edgetag.io/v1/oauth-app/client/token',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
      Authorization: `Basic ${btoa(clientId + ':' + clientSecret)}`,
    },
    body,
  },
)

const { access_token, refresh_token, expires_in } = await response.json()
// expires_in = 7200 (2 hours)
```

#### cURL

```bash
curl --request POST \
  --url https://api.edgetag.io/v1/oauth-app/client/token \
  --header 'Authorization: Basic {base64(client_id:client_secret)}' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data code={authorization_code} \
  --data code_verifier={verifier_plain_text} \
  --data grant_type=authorization_code
```

#### Response

```json
{
  "access_token": "kXzY5wUoUyt6jT_zlayTewPDtgGOuXFcrv0dqWSOBOo",
  "refresh_token": "PGFanyIh-GUKReZ5P1LlAsUJzBR76BQI5SJlIX4-aHQ",
  "expires_in": 7200
}
```

### 4. Refresh Access Token

Access tokens expire every **2 hours**. Refresh before expiry:

```
POST https://api.edgetag.io/v1/oauth-app/client/token
Content-Type: application/x-www-form-urlencoded
Authorization: Basic {base64(client_id:client_secret)}
```

| Field           | Value                                   |
| --------------- | --------------------------------------- |
| `refresh_token` | The refresh token from initial exchange |
| `grant_type`    | `refresh_token`                         |

#### JavaScript

```javascript
const body = new URLSearchParams({
  refresh_token: refreshToken,
  grant_type: 'refresh_token',
})

const response = await fetch(
  'https://api.edgetag.io/v1/oauth-app/client/token',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
      Authorization: `Basic ${btoa(clientId + ':' + clientSecret)}`,
    },
    body,
  },
)

const { access_token, expires_in } = await response.json()
```

#### Response

```json
{
  "access_token": "aYsxMIy3XguB9YORh2GzAL8E60MsVvXbG8IHxNFooSo",
  "expires_in": 7200
}
```

### 5. Revoke Token

Invalidate a user's access immediately:

```
POST https://api.edgetag.io/v1/oauth-app/client/revoke
Content-Type: application/x-www-form-urlencoded
Authorization: Basic {base64(client_id:client_secret)}
```

| Field             | Value                          |
| ----------------- | ------------------------------ |
| `token`           | The access token to invalidate |
| `token_type_hint` | `access_token`                 |

```bash
  --header 'Accept: application/json' \
```

```bash
    "sameSite": "Lax",
    "managed": true,
    "hostId": "",
    "teamId": "{team_id}",
    "cookieMapKey": "_shopify_y",
    "platform": "SHOPIFY"
  }'
```

### GET /tag/verify/mapping/{tagId}

Retrieve DNS records that need to be configured and check their validation status.

```bash
curl --request GET \
  --url https://api.edgetag.io/v1/tag/verify/mapping/{tagId} \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer {token}' \
  --header 'Team-Id: {team_id}'
```

**Response:**

```json
{
  "message": "...",
  "valid": false,
  "records": [
    {
      "type": "TXT",
      "name": "_cf-custom-hostname.bssql.mystore.com",
      "value": "a]2b3c4d-..."
    },
    {
      "type": "CNAME",
      "name": "bssql.mystore.com",
      "value": "bssql.customers.edgetag.io"
    }
  ]
}
```

**Note:** The exact record names and values vary per domain setup. The `records` array contains the precise values the customer must configure in their DNS provider. Always use these values directly -- do not fabricate or guess DNS record names.

| Field     | Type    | Description                                                           |
| --------- | ------- | --------------------------------------------------------------------- |
| `valid`   | boolean | `true` when all DNS records are properly configured                   |
| `records` | array   | Remaining records to configure. Already-validated records are omitted |

**Polling behavior:** Call this endpoint after the customer configures each record. Once `valid: true`, DNS setup is complete.

## Consent Configuration

### PATCH /tag/consent/{tagId}

Update consent mode on an existing tag. This is the preferred endpoint — simpler than `PUT /tag/{tagId}` since it only requires the consent field.

```bash
curl --request PATCH \
  --url https://api.edgetag.io/v1/tag/consent/{tagId} \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer {token}' \
  --header 'Team-Id: {team_id}' \
  --header 'Content-Type: application/json' \
  --data '{
    "consent": "CUSTOM"
  }'
```

### Consent Values

| Value      | Description                                                                    |
| ---------- | ------------------------------------------------------------------------------ |
| `DISABLED` | Consent checks are off — all events fire immediately                           |
| `CUSTOM`   | Your app manages consent. You control when `edgetag('consent', ...)` is called |

For white-label apps, use `**CUSTOM**` or `**DISABLED**`.

## Secret Encryption

All channel credentials **must be encrypted client-side** before sending to the API. EdgeTag uses hybrid AES + RSA encryption.

### Step 1: Get the Public Key

Call `GET /user/me` and extract the `publicKey` field. Each environment (sandbox/production) has a different key.

### Step 2: Encrypt

Uses AES for the secret value and RSA to protect the AES key:

```javascript
const CryptoJS = require('crypto-js')
const NodeRSA = require('node-rsa')
const { v4: uuidv4 } = require('uuid')

function generateAESKey() {
  const key = uuidv4() + uuidv4()
  const keyStr = key.substr(5, 43)
  const salt = key.substr(20, 22)
  const saltArray = CryptoJS.enc.Base64.parse(salt)

  return CryptoJS.PBKDF2(keyStr, saltArray, {
    keySize: 256 / 32,
    iterations: 1000,
  }).toString()
}

function encrypt(publicKey, value) {
  const message = value.trim()

  // AES encrypt the secret
  const aesKey = generateAESKey()
  const cypher = CryptoJS.AES.encrypt(message, aesKey)

  // Parse AES output
  const parsed = JSON.parse(
    cypher.toString(CryptoJS.format.OpenSSL)
      ? JSON.stringify({
          ct: cypher.ciphertext.toString(CryptoJS.enc.Base64),
          iv: cypher.iv.toString(),
          s: cypher.salt.toString(),
        })
      : '{}',
  )

  // RSA encrypt the AES key
  const rsa = new NodeRSA(publicKey)
  rsa.setOptions({ encryptionScheme: 'pkcs1', type: 'public' })
  const encryptedKey = rsa.encrypt(aesKey, 'base64')

  return {
    data: parsed.ct, // Encrypted secret value
    key: encryptedKey, // RSA-encrypted AES key
    iv: parsed.iv, // AES initialization vector
    salt: parsed.s, // AES salt
  }
}

// Usage:
const publicKey = '-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----'
const secret = encrypt(publicKey, 'my-api-key-value')
// Send secret.data, secret.key, secret.iv, secret.salt in the API request
```

**Working example:** [https://stackblitz.com/edit/edgetag-generator](https://stackblitz.com/edit/edgetag-generator)

### Step 3: Send in API Request

Include the encrypted fields in the `secrets` array:

```json
{
  "secrets": [
    {
      "variable": "API_KEY",
      "type": "PASSWORD",
      "optional": false,
      "data": "{encrypted_data}",
      "key": "{rsa_encrypted_aes_key}",
      "iv": "{initialization_vector}",
      "salt": "{salt}"
    }
  ]
}
```
